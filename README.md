# Proyecto Web Colaborativo

Este proyecto fue desarrollado por estudiantes de la carrera de **Arquitectura de Plataformas de Servicios Tecnológicos de la Información** en el **Instituto Superior Tecnológico Público "Daniel Alcides Carrión"**. Forma parte de las actividades prácticas del currículo, donde se aplican conocimientos en desarrollo web, control de versiones y trabajo en equipo.

## Descripción del Proyecto

El objetivo principal es crear una página web estática que integre múltiples módulos funcionales, utilizando tecnologías web estándar como HTML5, CSS3, JavaScript y PHP. El proyecto enfatiza la colaboración mediante GitHub, permitiendo a los estudiantes practicar flujos de trabajo profesionales en el desarrollo de software.

## Estructura del Proyecto

- `index.php` → Página principal con introducción al proyecto.
- `about.php` → Información detallada sobre el proyecto, objetivos y equipo.
- `contac.php` → Página de contacto con formulario interactivo.
- `register.php` → Formulario de registro de usuarios.
- `css/estilos.css` → Hojas de estilo para el diseño responsivo.
- `js/funciones.js` → Scripts para funcionalidades dinámicas.
- `img/` → Directorio de imágenes (logo y banner).

## Metodología de Desarrollo

El desarrollo se realizó siguiendo buenas prácticas de ingeniería de software:

1. **Planificación**: Definición de módulos y asignación de responsabilidades.
2. **Desarrollo Colaborativo**: Uso de GitHub para control de versiones.
   - Creación de un repositorio principal.
   - Cada integrante desarrolla su módulo en ramas separadas.
   - Revisiones de código y fusiones controladas.
3. **Pruebas**: Validación de funcionalidades y compatibilidad.
4. **Documentación**: Actualización continua del README y comentarios en el código.

### Pasos para Contribuir (Git Workflow)

1. Clonar el repositorio: `git clone <url-del-repositorio>`
2. Crear una rama para tu módulo: `git checkout -b feature/nombre-modulo`
3. Realizar cambios y commits: `git add .` y `git commit -m "Descripción del cambio"`
4. Subir cambios: `git push origin feature/nombre-modulo`
5. Crear Pull Request para revisión y fusión.

## Tecnologías Utilizadas

- **Frontend**: HTML5, CSS3, JavaScript
- **Backend**: PHP (para formularios)
- **Control de Versiones**: Git y GitHub
- **Herramientas**: Editor de código (VS Code), Navegadores web para pruebas

## Instalación y Ejecución

1. Clona el repositorio en tu máquina local.
2. Asegúrate de tener un servidor web con soporte PHP (como XAMPP o WAMP).
3. Coloca los archivos en el directorio raíz del servidor.
4. Accede a `index.php` desde tu navegador.
